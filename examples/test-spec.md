# Telegram Bot — Улучшения v2

## Обзор
Набор улучшений для существующего Telegram-бота GrandHub: система уведомлений, inline-режим и статистика использования.

---

## Фича 1: Система уведомлений

Добавить возможность отправки уведомлений пользователям по расписанию.

### Требования:
- Хранение расписаний в JSON-файле `data/schedules.json`
- Команда `/remind <время> <текст>` для создания напоминания
- Команда `/reminders` для просмотра активных напоминаний
- Фоновый процесс проверки расписаний каждую минуту
- Удаление напоминания после отправки

### Файлы:
- `src/commands/remind.ts` — обработчик команды /remind
- `src/commands/reminders.ts` — список напоминаний
- `src/services/scheduler.ts` — планировщик
- `src/types/schedule.ts` — типы

---

## Фича 2: Inline-режим

Реализовать inline-режим бота для быстрого поиска по базе знаний.

### Требования:
- Обработка `inline_query` от Telegram API
- Поиск по заголовкам статей в `data/articles.json`
- Возврат до 10 результатов в формате `InlineQueryResultArticle`
- Кэширование результатов на 60 секунд

### Файлы:
- `src/handlers/inline-query.ts` — обработчик inline запросов
- `src/services/search.ts` — сервис поиска
- `src/types/article.ts` — типы статей

---

## Фича 3: Статистика использования

Сбор и отображение статистики по использованию бота.

### Требования:
- Middleware для подсчёта команд по пользователям
- Хранение в `data/stats.json`
- Команда `/stats` — личная статистика пользователя
- Команда `/adminstats` — общая статистика (только для админов)
- Метрики: количество команд, активные пользователи, популярные команды

### Файлы:
- `src/middleware/stats-collector.ts` — сбор статистики
- `src/commands/stats.ts` — команда /stats
- `src/commands/adminstats.ts` — команда /adminstats
- `src/types/stats.ts` — типы статистики
